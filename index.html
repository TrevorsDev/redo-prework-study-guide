Git, CLI, SSH, and GitHub

    What is Git?

  • Git is a version control system that stores your code and its history, as well as separate versions of the code.
  • It allows you to create "save points" (called commits) of your work. It's good practice to commit your work whenever you reach a good stopping point. This creates a history of changes and allows you to revert to an earlier version if necessary.

    • CLI

    • How to navigate through directories and files, create a new folder
    • the ls command will allow us to see a list of files within our current directory
    • The pwd (print working directory) will print the current directory to the terminal
    • The cd command allows us to change directories cd .. will allow us to navigate backwards one directory at a time. and cd . will move you forward in a directory

    Do you have SSH keys? - GitHub

    How to Check

    • Enter ls -al ~/.ssh in terminal. Look for a public SSH key that ends in .pub.Ex: id_ed25519.pub
    • If an "error" occurs, you need ot generate a new SSH key pair

    Generating an SSH Key

    Generating a New SSH Key - GitHub
    • Heres a Steps Summary
      1. Enter ssh-keygen -t ed25519 -C “your_email@example.com” in your terminal to create a new SSH key.
      2. When prompted, save your key in the default file location and create a secure passphrase.

    Adding the New SSH Key to the ssh-agent

    Once you have your SSH keys, you can add them to the ssh-agent by reading How to Add SSH Key to ssh-agent - GitHub. The ssh-agent is a program that keeps track of a user's identity keys and passphrases.

    • Heres a Steps Summary
      1. Enter eval “$(ssh-agent -s)” in your terminal to start the ssh-agent.
      2. Add your private SSH key to the ssh-agent using the ssh-add command.

    Add the SSH Key to GitHub

    Read this article on how to set your Git username for every repository on your computer - GitHub

      You can add your newly generated SSH key to your GitHub account by reading this GitHub article

    • Heres a Steps Summary
    1. Copy the public SSH key to your clipboard. Be sure not to copy anything else after this point so that your SSH key stays on your clipboard!
    2. Navigate to the GitHub website and log in to your account.
    3. Click on Settings, and then click on SSH and GPG keys.
    4. Add your SSH key.

    Set Your Git Username for Every Repository on Your Computer

    • Git uses a username to associate any commits, or changes, to your code with an identity. The Git username is different from your GitHub username.
    • You can change the name that's associated with your Git commits using the git config command.
    • Changing the name associated with your Git commits using git config will only affect future commits and will not change the name used for past commits.

    Did you Connect it Successfully?

    • To test if you've set up your SSH connection correctly, enter ssh -T git@github.com into your terminal.
    • Verify that the fingerprint in the message matches the GitHub’s RSA public key fingerprint. If yes, you're connected.

    Version Control

    What is version control?

    • Version control is the practice of tracking changes to our code. With tracking, if we make changes to our code and decide that we don’t want to keep them or that they have a bug, we can easily revert to an earlier version of the code. Version control also makes our code more forgiving to mistakes because we can easily access and restore past versions.

      2. Identify where the bug happened in the workflow.

    Git, and Git Commands

    Git is the most widely used version control system, enabling tracking of changes to files and easier collaboration among multiple users.

  • Git status: checks what branch we are currently on
  • The git command must precede every Git statement.
  • The checkout command is to move the working branch to a new branch.
  • Git checkout -b branch-name: creates a new branch and switches to it.
  • The "branch name", ex: 'feature/starter-code' is the name of the new branch. We have added the prefix, feature, to indicate that it is a feature. It is good practice to name the branches for the feature that will be developed by them to help indicate the purpose of each branch.
  • To code to the current working branch, ex: 'feature/starter-code', we'll need to do the following:
    • git add -A
      • The add command adds modifications in the current working branch to the staging area.
      • The -A flag indicates that we want to add all changes.
    • Now that we have staged those files to be committed, let’s enter the git commit command with a commit message that describes what the commit contains. Ex:
      • git commit -m "add starter code"
    • Before we push this up to our repo, we should double-check if our local branch is in sync with the base branch in GitHub. So let’s pull down any changes from the base branch.:
      • git pull origin main
  • In Git, a commit is a snapshot of the changes made to the codebase at a specific point in time. A commit is a permanent and unchangeable record of the changes made to the codebase. A commit represents a single point in the history of the codebase.A stage, on the other hand, is a temporary holding area for changes that have been made to the codebase, but have not yet been committed. When you stage changes in Git, you are preparing them for a commit, but you have not yet made the commit. This allows you to review your changes and make sure they are correct before committing them.You can think of a commit as a final version of the changes and a stage as a draft version of the changes.
